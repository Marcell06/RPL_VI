  4. Sequence diagram itu untuk menggambarkan urutan proses yang terjadi pada sebuah halaman/form di aplikasi/sistem

  6. Yang ada di halaman atau form, dijabarkan urutan prosesnya di sequence diagram
  7. Bahan sequence diagram, yaitu deskripsi usecase + rancangan layar + class diagram
  8. Elemen inti pada Sequence Diagram:
    1. Actor = pengguna aplikasi/sistem
    2. Boundary = form/halaman di aplikasi
    3. Control = code program
    4. Entity = tabel
    5. Message = ativitas di aplikasi/sistem
  9. Aktor berinteraksi dengan boundary (yang ada halaman atau yang kita lihat sebagai pengguna) membuka aplikasi. Kemudian boundary berinteraksi dengan control memberikan instruksi bahwa aplikasi tersebut dibuka. Maka control akan menampilkan halaman yang dituju.
  10. Dalam sequence diagram bisa terdapat method, misalnya saat user login, maka C. Login akan select() data dari Entity Konsumen
